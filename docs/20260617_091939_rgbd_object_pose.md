# RGB-D Object 6D Pose

Goal: estimate object 6D poses for real-robot HumanEgo inference.

## Existing Project Support

The project has offline preprocessing code for this:

- `preprocess/DINOSAM.py`: object mask from RGB prompt
- `preprocess/DepthLifter.py`: lift 2D points/masks to 3D with depth and K
- `preprocess/OrientAnything.py`: estimate object frame with PCA/VLM

The inference template documents this path, but `run_inference.py` leaves the
real-time implementation as TODO.

## Online Module Added

Added:

```text
inference/object_pose_rgbd.py
```

Input:

```text
Frame(rgb, depth_m, K)
object_prompts: {obj1: "...", obj2: "..."}
anchor_key
```

Output:

```text
{obj_key: ObjectState(T_in_cam, kpts_local)}
```

Current method:

```text
DINO-SAM mask -> mask depth points -> 3D points in camera frame -> PCA pose
```

For `serve_bread`:

```text
obj1: "piece of bread ."
obj2: "a plate ."
anchor_key: "obj1"
pose_method: "pca1"
```

## Limits

- Requires aligned RGB-D and correct K.
- Requires valid depth on the object mask.
- Supports `pca1` / `pca2` online; VLM pose is not wired into online mode.
- `make_clean_image()` is still a placeholder and returns raw RGB.
