# Docs Standard

## Naming

All docs in this folder use:

```text
YYYYMMDD_HHMMSS_topic.md
```

Example:

```text
20260617_023904_serve_bread_reproduce.md
```

## Content Rules

- State the goal first.
- Use actual paths and commands.
- Keep steps executable.
- Avoid long background explanations.
- Record assumptions and current machine state when relevant.
- Prefer short sections and command blocks.

## Scope

Use `docs/` for project execution notes, reproduction steps, environment records,
debug notes, and decisions that should be easy to find later.
